# Рекомендации по реализации Среза №7: Переход на Next.js

Этот документ содержит рекомендации по переходу фронтенд-части Carpenter CRM на фреймворк **Next.js** в качестве **Среза №7**. Цель среза — заменить монолитный файл `index.html` структурированным, масштабируемым и визуально премиальным SPA-приложением на React с полной типизацией и сохранением всей существующей бизнес-логики Срезов 1-6.

---

## 1. Решение проблемы отсутствия Node.js

Для запуска и сборки Next.js проекта на компьютере разработчика обязательно требуется среда выполнения Node.js. Так как она сейчас не установлена, рекомендуется следующий сценарий установки:

1.  **Автоматическая установка через Windows Package Manager (winget):**
    Утилита `winget` уже встроена в Windows 10/11 и готова к работе. Мы можем установить официальную LTS (Long-Term Support) версию Node.js одной командой в PowerShell:
    ```powershell
    winget install OpenJS.NodeJS.LTS
    ```
2.  **Обновление переменных окружения (PATH):**
    После выполнения команды установщик пропишет пути к `node` и `npm` в системные переменные. Потребуется перезапустить терминал (или среду разработки), чтобы система распознала команды.
3.  **Верификация:**
    Убедиться в успешной установке с помощью команд:
    ```powershell
    node -v
    npm -v
    ```

## 2. Архитектура интеграции Next.js и FastAPI

При локальной разработке Next.js работает на порту `3000` (по умолчанию), а FastAPI — на порту `8000`. Чтобы связать их эффективно и безопасно, есть два пути:

*   **Вариант А (Рекомендуемый): Настройка Next.js API Rewrites**
    В файле `next.config.js` настраивается правило перенаправления: все запросы с фронтенда на `/api/:path*` будут проксироваться сервером Next.js на `http://localhost:8000/api/:path*`.
    *   *Плюсы:* Фронтенд обращается к `/api/...` на своем же домене/порту. Полностью отсутствуют проблемы с CORS (Cross-Origin Resource Sharing).
*   **Вариант Б: CORS на бэкенде**
    Добавление `CORSMiddleware` в FastAPI, разрешающего запросы с `http://localhost:3000`.
    *   *Минусы:* Требует изменения бэкенд-кода, а при деплое на продакшн потребуются разные настройки CORS для локального и боевого серверов.

## 3. Рекомендуемый стек технологий фронтенда

1.  **Базовый фреймворк:** Next.js (App Router). Это современный стандарт с автоматическим роутингом, оптимизацией и поддержкой Server/Client-компонентов.
2.  **Язык:** TypeScript. Позволит строго типизировать данные (Лиды, Заказы, Платежи) в соответствии с Pydantic-схемами бэкенда, что исключит опечатки в названиях свойств при рендеринге.
3.  **Стилизация (CSS):** CSS Modules. По регламенту проекта Tailwind CSS не используется без явного одобрения. CSS Modules позволяют писать чистый Vanilla CSS, изолируя стили каждого компонента в отдельном файле (например, `Sidebar.module.css`), что исключает коллизии классов.
4.  **Иконки:** Lucide React. Легковесный и современный набор SVG-иконок для панели навигации и кнопок действий.

## 4. Предлагаемая структура проекта (frontend)

После инициализации в папке `frontend` структура файлов будет выглядеть следующим образом:

```text
frontend/
├── next.config.js          # Конфигурация Next.js (прокси, реврайты)
├── package.json
├── tsconfig.json           # Настройки TypeScript
├── src/
│   ├── app/                # Маршрутизация и страницы (App Router)
│   │   ├── layout.tsx      # Главный каркас (шрифты, сайдбар, тема)
│   │   ├── page.tsx        # Главная страница (Финансовый дашборд директора)
│   │   ├── leads/          # Страница управления Лидами
│   │   ├── clients/        # Страница управления Клиентами
│   │   ├── orders/         # Страница управления Заказами и Календарем
│   │   ├── supply/         # Страница Снабжения (заявки, материалы, поставщики)
│   │   └── globals.css     # Глобальные CSS переменные и стили
│   ├── components/         # Многоразовые UI компоненты
│   │   ├── Sidebar/        # Боковое меню навигации
│   │   ├── UI/             # Кнопки, поля ввода, бейджи, модальные окна
│   │   └── Widgets/        # Финансовые карточки с градиентами
│   ├── services/           # Сервисы взаимодействия с API
│   │   └── api.ts          # API-клиент (fetch-обертки для CRUD)
│   └── types/              # Описания типов TypeScript
│       └── index.ts        # Интерфейсы Lead, Client, Order, Task, Payment и др.
```

## 5. Пошаговый план реализации Среза 7

Для поэтапного перехода предлагается следующий план:

### Шаг 1: Подготовка инфраструктуры
1.  Установить Node.js LTS на локальной машине с помощью `winget`.
2.  Переименовать текущую папку `frontend` в `frontend-old` (для резервного копирования).
3.  Создать новую папку `frontend` и инициализировать в ней проект Next.js с помощью `npx -y create-next-app@latest ./ --typescript --eslint --app --src-dir --no-tailwind --src-dir`.
4.  Настроить `next.config.js` для проксирования запросов к бэкенду на `http://localhost:8000`.

### Шаг 2: Дизайн-система и Базовый макет
1.  Перенести переменные цветов (HSL), шрифты (Inter/Outfit) и сброс стилей в `globals.css`.
2.  Создать компонент `Sidebar` (боковое меню) с переключением разделов и индикатором текущей роли.
3.  Оформить общий макет (`layout.tsx`), обеспечив адаптивность под мобильные устройства.

### Шаг 3: Модуль API и Типизация
1.  Описать интерфейсы TypeScript (`types/index.ts`) на основе Pydantic-схем бэкенда (`schemas.py`).
2.  Реализовать единый сервис `api.ts` с методами запросов:
    *   Лиды: `getLeads()`, `createLead()`, `deleteLead()`.
    *   Клиенты: `getClients()`, `convertLeadToClient()`, `deleteClient()`.
    *   Заказы & Задачи: `getOrders()`, `createOrder()`, `updateOrderStatus()`, `getTasks()`, `createTask()`, `updateTaskStatus()`.
    *   Снабжение: `getSuppliers()`, `getMaterials()`, `getSupplyRequests()`, `createSupplyRequest()`, `updateSupplyStatus()`.
    *   Финансы: `getFinancialSummary()`, `createPayment()`, `deletePayment()`.

### Шаг 4: Портирование экранов (Итеративно)
1.  **Главный дашборд:** Перенос 5 финансовых градиентных карточек с микро-анимациями на главную страницу `/`.
2.  **Модуль Лидов & Клиентов:** Раздельные страницы `/leads` и `/clients`. Замена добавления строк на всплывающие формы (модальные окна).
3.  **Модуль Заказов & Задач:** Страница `/orders` с вкладками управления заказами, списком задач и интерактивным таймлайном календаря.
4.  **Модуль Снабжения:** Страница `/supply` с вкладками для заявок на закупку, справочника материалов и списка поставщиков.
5.  **Финансовый учет:** Детальная таблица доходов и расходов с историей платежей и роль-гардом директора.

## 6. Рекомендации по UX/UI (Premium Aesthetics)

*   **Glassmorphism & Shadows:** Заменить грубые рамки карточек на полупрозрачные фоны с эффектом `backdrop-filter: blur(10px)` и мягкими тенями `box-shadow`.
*   **Микро-анимации:** Использовать переходы `transition: all 0.2s cubic-bezier(0.4, 0, 0.2, 1)` для кнопок, вкладок и строк таблиц при наведении.
*   **Скелетоны (Skeleton Loading):** Вместо простого текста "Загрузка..." показывать анимированные серые блоки-заглушки на время ожидания ответов от API.
*   **Всплывающие уведомления (Toasts):** Использовать аккуратные всплывающие сообщения об успехе/ошибке в углу экрана вместо изменения текста статуса в верстке.
