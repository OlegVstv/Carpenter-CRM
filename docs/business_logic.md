Internal CRM для столярного производства 1. Общая концепция продукта Internal CRM — это внутренняя операционная система компании, разработанная под реальные процессы предприятия. Система предназначена для управления: клиентами и лидами; заказами и проектами; задачами сотрудников; снабжением и материалами; документами; контролем сроков; финансовыми статусами; управленческой аналитикой; единым источником данных для AI-агентов. Система проектируется как простая, понятная и масштабируемая. На старте — для команды 10–15 человек, с возможностью дальнейшего расширения. 2. Формат размещения Система работает локально на собственном сервере компании в офисе. Доступ сотрудников осуществляется через веб-интерфейс по локальной сети. При необходимости в будущем возможен защищенный внешний доступ через VPN. При масштабировании возможна разработка приложения. На текущий момент работа через браузер (с адаптацией под mobile разрешение экрана) 3. Базовая архитектура системы На локальном сервере размещаются: backend; frontend; центральная база данных; файловое хранилище; каталог компании; история действий; интеграционные модули; AI-ready слой. Пользователи системы (на момент разработки): генеральный директор; операционный директор; начальник цеха; менеджер по работе с клиентами; снабженец; бухгалтер; администратор. 4. Рекомендуемый стек разработки Backend: Python FastAPI Frontend: React / Next.js Database: PostgreSQL Auth: роли и права доступа Deployment: Docker Compose Backup: ежедневное резервное копирование 5. AI-Ready архитектура Система проектируется с готовностью к AI-интеграции. Предусматривается: структурированная база данных; единый каталог знаний; API-доступ для AI; разграничение прав доступа; история действий; источники данных для AI-агентов. 6. Основные модули CRM Модуль 1. Главный каталог компании Центральная база знаний и внутренних данных. Содержит: должностные инструкции; регламенты; шаблоны договоров; шаблоны КП; скрипты продаж; инструкции сотрудников и обучение; прайсы; базу продукции; базу поставщиков; базу материалов; FAQ; архив решений; источники данных для AI. Модуль 2. CRM / Лиды / Клиенты Функции: входящий лид через звонок / WhatsApp / вручную менеджером; создание карточки клиента; контакты; история общения; ответственный менеджер; комментарии; задачи по клиенту. Дополнительно предусмотреть поле: источник заявки. Варианты источников: звонок; WhatsApp; Instagram; сайт; рекомендация; повторный клиент; реклама; вручную внесено. Это позволит в будущем автоматизировать источники и вести аналитику каналов продаж. Статусы клиента / сделки: новый запрос; просчет / КП; КП отправлено; согласование; предоплата; передан в производство; оплачен полностью; закрыт; отказ; архив. Модуль 3. Проекты / Заказы клиентов Ключевой рабочий модуль. Карточка заказа включает: изделие / тип работ; ТЗ; сроки; стоимость; оплаты; статус производства; статус закупки; статус доставки; ответственных; документы; фото; комментарии; историю изменений. Дополнительно включить календарный блок проекта: дата замера; дата выезда; дата доставки; дата монтажа; контрольные даты; напоминания ответственным. Статусы заказа: принят; ожидает оплаты; согласован; передан в производство; ожидает материалы; в производстве; готов; на доставке; сдан клиенту; закрыт; приостановлен; проблемный. Важно: при разработке UI/UX визуальные статусы заказов должны быть реализованы максимально наглядно. Например: цветовые индикаторы; Kanban-доска; таблица со статусами; сигналы просрочки; сигналы критичных заказов. Это позволит мгновенно видеть состояние заказов и узкие места. Модуль 4. Задачник / Планировщик Функции: личные задачи; задачи от руководителя; задачи по проектам; дедлайны; календарь; напоминания; приоритеты; комментарии; чек-листы. При масштабировании все модули имеющие сроки (задачи, проекты и т.д.) могут быть интегрированы с Google Calendar API для подключения через OAuth 2.0 со своим аккаунтом Google, чтобы все сроки отражались в личном календаре пользователя. Модуль 5. Чат / Комментарии На старте: комментарии внутри карточек; упоминания сотрудников; уведомления; история обсуждений; вложения. При масштабировании можно сделать полноценный внутренний чат. Модуль 6. Снабжение / Материалы Функции: каталог материалов; поставщики; потребности по заказам; заявки на закупку; статусы закупок; сроки поставки; комментарии. Модуль 7. Финансовый light-блок По заказам: номер заказа; сумма; предоплата; остаток; долг; статус оплаты; факт поступления. По закупкам: номер заказа; сумма закупки; статус; согласование. Модуль 8. Документы Хранение: договоров; КП; счетов; актов; спецификаций; ТЗ; фото; чертежей. Модуль 9. Дашборд руководителя Показывает: заказы в работе; просроченные заказы; ожидаемые оплаты; ожидаемые закупки; что в производстве; что готово к сдаче; просроченные задачи; клиенты на стадии КП; суммы поступлений и расходов; риски. Модуль 10. AI-ready слой Будущие роли: AI-секретарь директора; AI-коммерческий помощник; AI-юрист; AI-снабженец; AI-аналитик; AI-маркетолог. 7. Минимальная версия MVP На старте достаточно: пользователи и роли; клиенты и лиды; источник заявки; проекты / заказы; визуальные статусы заказов; календарь замеров / доставок / монтажей; задачи и календарь; снабжение; документы; финансовый light-блок; дашборд директора; AI-ready API слой. 8. Этапы разработки Этап 1. Проектирование Процессы, роли, статусы, структура данных, UI/UX прототипы. Этап 2. MVP CRM Лиды, клиенты, проекты, задачи, документы. Этап 3. Операционные блоки Снабжение, финансы, визуальные статусы, календари. Этап 4. AI-ready подготовка API, база знаний, источники данных. Этап 5. Тестирование Работа на реальных заказах, корректировки, обучение. Этап 6. Масштабирование – отдельный проект Новые модули и новые направления бизнеса. 9. Итоговая формулировка Internal CRM — это локальная AI-ready операционная система для малого производственного бизнеса, объединяющая клиентов, заказы, задачи, снабжение, документы, статусы и контроль в единой базе данных, с возможностью дальнейшего масштабирования без смены платформы.
